How this works

  • Each line is target_field = $.json.path — the path is resolved against the raw payload the connector ingested.
  • entity_ref is a special target field: if present, it's what Triggers group repeated events by (e.g. per-customer) instead of per-record.
  • A missing/unmatched path yields null for that field rather than failing normalization — safe to map speculatively.
  • Optional fingerprint fields suppress exact duplicates before they retrigger analysis and response. Raw records remain available for evidence.
  • Every create is written to an immutable audit log for compliance review.
