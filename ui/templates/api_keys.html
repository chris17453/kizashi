About API keys

  • Each key authenticates one connector/agent at POST /v1/ingest via the X-Api-Key header.
  • The plaintext key is shown exactly once, right after creation — copy it immediately. Only its hash is ever stored, so it can't be retrieved again later.
  • Revoking a key takes effect immediately; a revoked key's connector will start getting 401s on its next ingest attempt.
  • Every create/revoke is written to an immutable audit log for compliance review.
