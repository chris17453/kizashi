How this works

  • v1 only enforces Raw — Normalized/Event policies can be configured but the scheduled sweep doesn't act on them yet.
  • Archived data is re-ingestible through the same pipeline — the archive format preserves enough fidelity to replay ingestion → normalization → analysis.
  • Disabling a policy stops the sweep from archiving/deleting that data class without losing the configured TTL.
  • Every create/update is written to an immutable audit log for compliance review.
