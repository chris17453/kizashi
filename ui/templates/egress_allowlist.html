How this works

  • This is a replace-the-whole-list save, not a per-domain add/remove — the list is small and operator-managed.
  • Egress Gateway tunnels HTTPS traffic byte-for-byte after the initial CONNECT — it never decrypts or inspects request paths or bodies, only the destination host is checked and logged.
  • Denied destinations still show up in the audit log (allowed=false), which is useful for spotting a connector that's misconfigured or reaching somewhere unexpected.
  • Connector/action adoption of the proxy is opt-in per process (EGRESS_PROXY_URL) — this allowlist only has an effect on traffic that's actually routed through the gateway.
